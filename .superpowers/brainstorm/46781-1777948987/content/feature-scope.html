What's the scope for Phase 1?

We already have the backend APIs to power most of this. The question is how much to build in the first mobile release.

A. Full dashboard on mobile

Calendar + appointments + CRM + services + staff + finances — essentially the complete dashboard but mobile-first

B. Essentials first

Calendar + appointments + notifications — the things staff/masters need while away from the desk. Everything else comes in Phase 2

C. Hyper-focused MVP

Appointments list + confirm/cancel + push notifications. Nothing else. Validate that people actually use the mobile app before building more
