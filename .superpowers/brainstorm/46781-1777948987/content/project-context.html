Beautica Marketplace

Existing platform → React Native app brainstorm

🖥️ Current Web App

  • Public: Search salons, browse services, guest booking
  • Client: OTP auth, "My Appointments", notifications (SSE)
  • Salon Dashboard: Calendar, staff, services, CRM, analytics
  • Master Cabinet: Personal appointments, clients, services, finances

⚙️ Tech Stack

  • Backend: Go 1.24, Fx DI, GORM, PostgreSQL
  • Frontend: React + TS + Vite + MUI + Redux Toolkit
  • Auth: OTP (SMS/Telegram) + JWT (2h access / 30d refresh)
  • Integrations: 2GIS API (search, geocoding, places)
Monorepo REST API SSE Streaming

👥 User Roles

  • Guest: Browse & book without account
  • Client: Registered user with bookings
  • Master (individual): Private practice CRM
  • Salon Staff: Owner, Admin, Receptionist

First Question: Who is the RN app for?

The existing web platform serves very different user types with very different needs. Before we design architecture, we need to decide which users the mobile app targets first.

Should the RN app be:

  • A client-facing app — browse salons, book appointments, manage bookings (like a mobile version of the public SPA)
  • A salon staff app — calendar, appointments, CRM on-the-go (mobile dashboard)
  • A master (individual) app — private practice management for solo beauticians
  • All of the above — one app with role-based screens (most ambitious)
