# Интеграция функционала (Записи, Клиенты, Услуги, Календарь) в Кабинет Мастера

Данный план описывает архитектурный и продуктовый подход к внедрению функционала управления бизнесом в "Кабинет Мастера" (`/master-dashboard`).

## Отличия контекстов (Master vs Salon)

Ключевая логика: **Мастер является самостоятельной бизнес-единицей (Solo Master)**, которая дополнительно может быть сотрудником в одном или нескольких Салонах.
- У Мастера есть **свои личные** Услуги, Специализации, Клиенты и Записи. Они никак не пересекаются с услугами и специализациями Салонов, в которых он работает.
- Находясь в Кабинете Мастера, пользователь управляет своим *личным* бизнесом, а также видит агрегированную информацию (Записи, Клиенты) из Салонов, в которых он состоит.

## Proposed Changes

### 1. Блок «Записи» (Appointments)

**Frontend:**
- [NEW] `frontend/src/pages/master-dashboard/ui/MasterDashboardAppointments.tsx`
- Переиспользование табличного компонента из Дашборда Салона, но с фильтром по **"Салонам"** (опции: Все, Только Мои (личные записи), [Список Салонов]).
- **Создание записи:** Из Кабинета Мастера можно создавать **только Личные** записи. Кнопка "Добавить запись" открывает дровер создания записи, который работает исключительно в контексте "Solo Master" (выбор личных услуг, личных клиентов). Для создания записи в рамках салона мастер должен перейти в Кабинет Салона.
- **Редактирование записи:** Если запись "Личная", мастер может ее полноценно редактировать. Если запись "Салонная", он может менять ее статус (если есть права) или видеть в read-only режиме.

**Backend:**
- [MODIFY] `ListAppointments` (`backend/internal/service/master_dashboard.go`) — добавить поддержку серверной пагинации (`page`, `pageSize`) и возвращать как салонные записи (где мастер = текущий пользователь), так и личные записи мастера.
- [NEW] Эндпоинты для создания и редактирования *личных* записей мастера (без привязки к `salon_id`).

---

### 2. Блок «Клиенты» (Clients)

**Frontend:**
- [NEW] `frontend/src/pages/master-dashboard/ui/MasterClientsListView.tsx`
- Data Grid со списком клиентов. Те же фильтры, что и в Салоне, но фильтр "Сотрудники" заменен на фильтр **"Салоны"** (Все, Только Мои, [Список Салонов]).
- "Только Мои" отображает клиентов, которых мастер завел лично (Solo Master clients).
- При выборе конкретного салона отображаются клиенты этого салона (если у мастера есть к ним доступ, либо те, с кем у него были записи).

**Backend:**
- [NEW] Эндпоинты для управления **личными клиентами** мастера (`master_clients` или `clients` с `master_id` вместо `salon_id`).
- [NEW] `GET /api/v1/master-dashboard/clients` — метод агрегации. Возвращает личных клиентов мастера + уникальных клиентов из салонов (по которым у мастера были записи).

---

### 3. Блок «Услуги и Специализации» (Services)

Мастер может создавать и редактировать собственные услуги и специализации. Они не пересекаются со специализациями мастера внутри Салона (например, в салоне он Стилист, а лично — Барбер и Бровист).

**Frontend:**
- [NEW] `frontend/src/pages/master-dashboard/ui/MasterDashboardServices.tsx`
- Страница управления **Личными Услугами** мастера. Аналогична `ServicesView` салона, но работает с контекстом профиля мастера.
- В профиле мастера (`ProfileSection`) управление специализациями сохраняется только для его **личного** профиля.

**Backend:**
- [NEW] Таблицы/сущности для личных услуг мастера (например, `master_services`). Если текущая БД уже поддерживает услуги без `salon_id` (привязанные к `master_profile_id`), использовать их.
- [NEW] Управление CRUD операциями для личных услуг (`GET/POST/PUT/DELETE /api/v1/master-dashboard/services`).

---

### 4. Блок «Календарь» (Calendar)

Календарь Мастера фокусируется на его личном расписании, объединяя личные дела и работу в салонах.

**Frontend:**
- [NEW] `frontend/src/pages/master-dashboard/ui/MasterDashboardCalendar.tsx`
- **Режим "Неделя" / "Месяц"**: Ивенты (записи) раскрашиваются в зависимости от источника: "Личные" одним цветом, записи из "Салона А" — другим, из "Салона Б" — третьим.
- **Режим "День"**: Одна единственная колонка (таймлайн мастера). Все записи из разных салонов и личные записи выводятся на одном таймлайне.
- При клике на пустую ячейку открывается `CreateAppointmentDrawer` для создания **только Личной** записи.

**Backend:**
- [NEW] `GET /api/v1/master-dashboard/schedule` — возвращает сводное расписание мастера (рабочие часы, перерывы, выходные). Расписание строится на основе его личных настроек графика + графиков из салонов.

## Verification Plan

### Automated Tests
- Запуск backend-тестов `go test ./internal/service/...` после добавления/изменения эндпоинтов (особенно важно покрытие агрегации записей и клиентов).
- Запуск линтера фронтенда `npm run lint`.

### Manual Verification
- Авторизоваться как Мастер, состоящий в Салоне А.
- **Услуги**: Создать личную услугу. Убедиться, что она не появилась в Салоне А.
- **Специализации**: Изменить личную специализацию. Убедиться, что специализация мастера внутри Салона А осталась прежней.
- **Записи и Календарь**: Создать личную запись из Кабинета Мастера. Убедиться, что она отображается в его календаре, но не видна в календаре Салона А.
- **Клиенты**: Проверить работу фильтра "Салоны" (Все, Только Мои, Салон А). Убедиться, что при "Только Мои" видны только клиенты личных записей мастера.
