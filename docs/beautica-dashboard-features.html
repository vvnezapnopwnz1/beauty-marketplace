Критика текущего дашборда

🔴
Критично: нет ценности для бизнеса
Dashboard сейчас — пустая страница. Владелец салона не получает ни одной полезной цифры. Без реального функционала салоны не будут регистрироваться. Это blocker для монетизации.
🔴
Критично: записи не управляются
Appointment модель в БД есть, но нет API для владельца: нельзя видеть записи, менять статус (confirm/cancel), управлять расписанием. Гостевая запись уходит "в пустоту".
🟡
Важно: нет календаря
Все конкуренты строятся вокруг календаря. Без визуального расписания по дням/часам невозможно управлять загрузкой салона. Это core feature номер один.
🟡
Важно: нет клиентской базы
Даже простой CRM нет: кто записывался, сколько раз, какие услуги. Для удержания клиентов это must have. Данные уже частично есть в appointments — нужно агрегировать.

Ключевые выводы

🇷🇺
Рынок РФ отстаёт
YCLIENTS и DIKIDI — B2B-инструменты без маркетплейса. У них нет AI-фич, нет unified inbox, нет динамического ценообразования. Они застряли в парадигме 2018 года. Beautica может перескочить сразу в 2026.
🤖
AI — главный разрыв
Только Zenoti имеет глубокий AI (Zeenie). На российском рынке ноль конкурентов с AI. No-show предикция, AI-ресепшн, smart scheduling — всё свободно.
💬
Мессенджеры = Россия
В РФ Telegram и WhatsApp — основные каналы связи. Ни YCLIENTS, ни DIKIDI не дают unified inbox для бизнеса. Mangomint Connect (только США) показывает спрос на такую фичу.
⚡
Маркетплейс + SaaS = редкость
Большинство платформ — либо маркетплейс (Booksy), либо SaaS (YCLIENTS). Beautica может стать единственным, кто совмещает: клиент находит салон и салон управляет бизнесом в одном месте.

Core — минимум для работающего дашборда

Без этих фич дашборд бесполезен. Это то, что есть у всех конкурентов — паритет рынка.

1. Живой календарь записей

MVP Блокер

Визуальный календарь с записями по дням, часам и мастерам. Drag & drop для перемещения. Цветовая индикация статусов.

Backend (новые эндпоинты)

  • GET /api/v1/dashboard/appointments?from=&to=
  • PATCH /api/v1/dashboard/appointments/{id}/status
  • PUT /api/v1/dashboard/appointments/{id} (перенос)
  • POST /api/v1/dashboard/appointments (ручная запись)

Frontend

  • Вид: день / неделя / месяц
  • Колонки по мастерам (staff)
  • Статусы: pending → confirmed → completed / cancelled / no_show
  • Click на слот → создать запись

2. Управление записями

MVP Блокер

Список всех записей с фильтрами, изменение статуса, детали клиента. Без этого гостевые записи уходят в никуда.

Что нужно

  • Таблица записей с пагинацией и фильтрами
  • Быстрые действия: подтвердить / отменить / завершить
  • Детали: клиент, услуга, мастер, время, заметка
  • Push/Telegram уведомление при новой записи

Готовность

  • ✅ Модель Appointment — есть
  • ✅ Статусы (enum) — определены
  • ✅ POST bookings — работает
  • ❌ GET/PATCH для dashboard — нет

3. Управление услугами

MVP

CRUD для услуг салона: название, длительность, цена, сортировка, активность. Без этого клиент не знает что бронировать.

Backend

  • GET /api/v1/dashboard/services
  • POST /api/v1/dashboard/services
  • PUT /api/v1/dashboard/services/{id}
  • DELETE /api/v1/dashboard/services/{id}

Frontend

  • Таблица услуг с inline-edit
  • Drag-sort для порядка отображения
  • Toggle активности
  • Категории услуг (группировка)

4. Управление мастерами

MVP

Список мастеров, их расписание, привязка услуг к конкретным мастерам.

Backend

  • GET/POST/PUT/DELETE /api/v1/dashboard/staff
  • GET/PUT /api/v1/dashboard/staff/{id}/schedule
  • Привязка staff ↔ services (M:N)

Что видит владелец

  • Карточки мастеров с фото и статусом
  • Расписание каждого мастера (WorkingHour)
  • Какие услуги оказывает
  • Статистика по мастеру (позже)

5. Расписание салона

MVP

Рабочие часы по дням недели, выходные, праздники. WorkingHour уже в БД — нужен только UI и API.

Backend

  • GET /api/v1/dashboard/schedule
  • PUT /api/v1/dashboard/schedule
  • Поддержка valid_from / valid_to для сезонных графиков

Frontend

  • Сетка ПН–ВС с opens_at / closes_at
  • Исключения (праздники, отпуск)
  • Применить расписание к конкретному мастеру

6. Виджеты главного экрана

MVP

При входе в Dashboard — сразу ключевые цифры: записи сегодня, новые записи, выручка, загрузка.

Виджеты (первая версия)

  • 📅 Записей сегодня: X (из них подтверждённых Y)
  • 🆕 Новых записей за неделю: +N
  • 👥 Всего клиентов: N
  • ⭐ Средний рейтинг: 4.8

Backend

  • GET /api/v1/dashboard/stats
  • Агрегация по appointments за период
  • Подсчёт уникальных клиентов
  • Средний рейтинг из reviews (позже)

Smart фичи — выше паритета

Эти фичи делают дашборд умнее чем у YCLIENTS/DIKIDI. Реализуемы за 1–3 недели каждая.

7. CRM: клиентская база

Скоро Конкурентное преимущество

Профили клиентов с историей визитов, предпочтениями, заметками. Данные собираются из appointments автоматически.

Что показывать

  • Имя, телефон, дата первого визита
  • Количество визитов, LTV
  • Любимые услуги, любимый мастер
  • Последний визит + когда ожидаем следующий
  • Заметки мастера (формулы красок, аллергии)

Чего нет у конкурентов

  • ❌ YCLIENTS: CRM слабый, нет прогноза
  • ❌ DIKIDI: только история записей
  • ✅ Beautica: можем добавить предикцию «когда клиент вернётся» по истории визитов

8. Умные уведомления (Telegram-first)

Скоро Уникально для РФ

Система уведомлений через Telegram Bot API — дешевле SMS, выше открываемость (98% vs 20% email). Это уже в плане проекта!

Для владельца

  • 🔔 Новая запись → Telegram мгновенно
  • ⚠️ Отмена → уведомление + предложение заполнить слот
  • 📊 Ежедневная сводка: записи на завтра
  • 💰 Еженедельный отчёт: выручка, загрузка

Для клиента

  • ✅ Подтверждение записи
  • ⏰ Напоминание за 24ч и за 2ч
  • 🌟 Просьба оставить отзыв после визита
  • 🎁 Персональные предложения

9. Аналитика загрузки

Скоро

Графики: загрузка по дням недели, пиковые часы, самые популярные услуги, конверсия записей.

Графики и метрики

  • 📊 Загрузка по часам (heatmap)
  • 📈 Тренд записей за 30 дней
  • 🏆 Топ-5 услуг по выручке
  • 👤 Топ мастеров по загрузке
  • 🔄 Процент возврата клиентов

Реализация

  • Всё считается по таблице appointments
  • SQL агрегация + кэш 1 час
  • Библиотека: Recharts (уже в стеке React)
  • Один endpoint: GET /api/v1/dashboard/analytics

10. Быстрые действия (Command Center)

Скоро

На главной дашборда — actionable карточки: «3 записи ждут подтверждения», «Мастер X не работает завтра — 5 записей», «Клиент Y не приходил 60 дней».

Типы карточек

  • 🟡 Записи требующие подтверждения
  • 🔴 Конфликты расписания
  • 💤 «Спящие» клиенты (давно не были)
  • ⚡ Свободные слоты сегодня (потеря выручки)
  • ⭐ Новые отзывы без ответа

Почему это сильно

  • YCLIENTS: только сухие таблицы
  • DIKIDI: только уведомления
  • Beautica: проактивные подсказки — дашборд говорит владельцу что делать

11. Отзывы и репутация

Скоро

Модерация и ответы на отзывы. Схема Review уже в БД — нужен API и UI. Верифицированные отзывы (только после записи) — конкурентное преимущество.

12. Профиль салона (редактирование)

Скоро

Редактирование публичного профиля: фото, описание, контакты, категория. Сейчас профиль формируется из 2GIS — нужна возможность «заклеймить» и дополнить.

Уникальные фичи — то, чего нет у конкурентов в РФ

Именно эти фичи создают «вау-эффект» и привлекают салоны на платформу. Ни одна из них не реализована на российском рынке.

13. 🤖 AI-ассистент «Beautica AI»

Глубокая фича 🏆 Главный дифференциатор

Чат-бот в дашборде который отвечает на вопросы владельца на естественном языке: «Какой мастер загружен меньше всего на этой неделе?», «Покажи выручку за март», «Какие клиенты давно не были?»

Что умеет

  • NL-запросы к данным салона
  • Рекомендации по расписанию
  • Подсказки по маркетингу
  • Генерация текста для акций
  • Ответы на отзывы

Реализация

  • LLM API (Claude/GPT) + RAG по данным салона
  • Стоимость: ~$0.01–0.05 за запрос
  • Можно сделать в платном тарифе
  • MVP: 5 фиксированных типов запросов

14. 🔮 No-Show предикция

Глубокая фича 🏆 Нет на рынке РФ

ML-модель предсказывает вероятность неявки по истории клиента, дню недели, времени, погоде. Автоматически предлагает действия: подтверждение, предоплата, overbooking.

Сигналы для модели

  • История no_show клиента
  • День недели + время
  • Интервал до записи (заранее vs день-в-день)
  • Способ записи (онлайн vs ручная)
  • Погода (опционально, API)

Действия

  • 🟢 Низкий риск → обычное напоминание
  • 🟡 Средний → двойное подтверждение
  • 🔴 Высокий → предоплата / overbooking
  • Статистика: Zenoti даёт −40% no-show

15. 💬 Unified Inbox (Telegram + WhatsApp)

Глубокая фича 🏆 Нет аналога в РФ

Все сообщения от клиентов — Telegram, WhatsApp, чат на сайте — в одном окне. Привязка к профилю клиента. Быстрые ответы, шаблоны.

Как это работает

  • Telegram Bot API → webhook → наш backend
  • WhatsApp Business API (через провайдера)
  • Каждое сообщение привязано к client_user_id
  • Владелец отвечает из дашборда

Аналог: Mangomint Connect

  • Mangomint Connect = звонки + SMS + чат
  • $75/мес — дорого для РФ
  • Beautica: Telegram-first (бесплатный API)
  • WhatsApp: через Wazzup/Chat-API ($10–30/мес)

16. ⏰ Smart Waitlist (авто-замена отмен)

Глубокая фича 🏆 Нет на рынке РФ

Клиент отменил → система автоматически предлагает освободившийся слот тем, кто в waitlist на эту услугу/мастера/день. Модель WaitlistEntry уже в БД!

Логика

  • Клиент отменяет → trigger
  • Ищем waitlist записи matching (service, staff, date range)
  • Отправляем push/Telegram: «Слот освободился! Записаться?»
  • Первый кто нажал «Да» — получает слот
  • Timeout 30 мин → следующий в очереди

Ценность

  • −50% пустых слотов от отмен
  • Клиенты довольны (получают слот быстрее)
  • Салон не теряет выручку
  • Fresha только в 2026 это планирует!

17. 💰 Динамическое ценообразование

Глубокая фича Осторожно: деликатная тема

Автоматические скидки в непиковые часы, повышение в пиковые. Не «surge pricing» — а «умные акции»: «Стрижка в ПН–СР до 12:00 со скидкой 20%».

Подход для РФ

  • Позиционировать как «акции», не «surge pricing»
  • Владелец задаёт % скидки для off-peak
  • Система подсказывает оптимальные часы по данным
  • Бейдж «🔥 -20% до 12:00» на клиентской стороне

Результат

  • +15–25% загрузка в непиковые часы
  • Клиенты получают скидку
  • Салон заполняет пустые слоты
  • Win-win для обеих сторон

18. 📸 Портфолио мастера (до/после)

Скоро Визуальный вау

Мастер загружает фото работ с привязкой к услуге. На клиентской стороне — галерея «До/После». GlossGenius показал что это сильно конвертит.

19. 🎯 Маркетинг-автоматизация

Глубокая фича Монетизация

Автоматические Telegram-кампании по триггерам: клиент не был 30 дней → «Скучаем! Скидка 15%». День рождения → поздравление + предложение. Первый визит → «Как вам? Оставьте отзыв».

Триггеры

  • ⏰ Клиент не был N дней
  • 🎂 День рождения
  • 👋 Первый визит завершён
  • ⭐ Нет отзыва после визита
  • 📅 Регулярный клиент — пора записаться

Канал доставки

  • Telegram Bot API (бесплатно)
  • WhatsApp Business (по подписке)
  • Push-уведомления (когда будет PWA/app)
  • Шаблоны сообщений с персонализацией

20. 🏆 Программа лояльности

Глубокая фича

Бонусные баллы за визиты, реферальная программа, VIP-статусы. DIKIDI делает это, но примитивно. Мы можем сделать с геймификацией: бейджи, уровни, челленджи.

Фичи для роста платформы

Эти фичи увеличивают ценность платформы и помогают с монетизацией.

21. Статистика из маркетплейса

Скоро Маркетплейс-only

Только маркетплейс может дать это: сколько людей видели карточку салона, сколько кликнули, конверсия в запись. YCLIENTS этого не может — они не маркетплейс.

Метрики

  • 👁️ Просмотры карточки в поиске
  • 👆 Клики на профиль

Фаза 1 — 2–3 недели
Рабочий дашборд (Core MVP)
Блокер Backend API для dashboard: CRUD записей, услуг, мастеров, расписания. Виджеты главного экрана. Визуальный календарь (день/неделя). Управление статусами записей. Без этого салоны не могут работать на платформе.

Фаза 2 — 2–4 недели
Умный дашборд (Smart)
Telegram-уведомления (для владельца + клиента). CRM с историей клиентов. Аналитика загрузки (Recharts). Command Center (проактивные подсказки). Отзывы. Редактирование профиля.

Ключевое: Telegram Bot API интеграция = 2–3 дня. CRM собирается из существующих appointments.
Фаза 3 — 3–6 недель
Уникальные фичи (Дифференциация)
Smart Waitlist (авто-замена отмен). Unified Inbox (Telegram + WhatsApp). Маркетинг-автоматизация (триггерные кампании). Портфолио мастеров (до/после). Маркетплейс-аналитика (воронка показы→записи).

Ключевое: WaitlistEntry уже в БД. Telegram Bot API уже нужен для фазы 2 — расширяем.
Фаза 4 — 4–8 недель
AI и монетизация
AI-ассистент (NL-запросы к данным). No-show предикция. Динамическое ценообразование. Программа лояльности. Онлайн-оплата (ЮКасса/СБП). Приоритет в выдаче (paid). Мульти-локации.

Ключевое: AI-ассистент MVP = обёртка над LLM API + SQL-запросы. No-show предикция начинается с простых правил (history-based), потом ML.
