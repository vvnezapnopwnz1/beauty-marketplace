UX улучшения
Конкретные изменения с impact
Отсортированы по соотношению «усилие / визуальный эффект». Первые 4 — Quick Wins, остальные — следующая итерация.
⚡ 30 минут

Горизонтальный скролл категорий

«Эпиляция» переносится на вторую строку и нарушает визуальную линию. Решение: overflow-x: auto + white-space: nowrap + скрыть scrollbar. Все 9 категорий — одна строка.

→ Устраняет визуальный баг, делает UI чище
⚡ 1–2 часа

FAB «На карте» — fixed bottom-center

Кнопка сейчас в строке фильтров — смотрится как активный фильтр, путает. Перенести в position: fixed; bottom: 24px; left: 50%. Та же кнопка, новое место — другой UX.

→ Исправляет критическую путаницу «фильтр vs режим»
⚡ 30 минут

Убрать список под картой

«Ближайшие к вам» под картой создаёт два параллельных режима одновременно. В map-режиме должна быть только карта. Список — только через кнопку «Список».

→ Убирает UX-путаницу двойного режима
⚡ 15 минут

Обрезать длинные адреса

«Белоозёрский (Воскресенск городской округ, Московская обл.) Молодёжная улица, 1к» — переполняет карточку. Нужен -webkit-line-clamp: 2 с ellipsis. Адрес — вспомогательная инфо.

→ Карточки читаются, контент не перегружает
🔧 2–4 часа

Featured-карточка с реальным контентом

Сейчас featured больше по размеру, но показывает то же самое. Нужно: услуги с ценами из item.services[], кнопка «Записаться» если onlineBooking: true, категория-бейдж из rubricNames. Для plain 2GIS — хотя бы 3 тега рубрик.

→ Визуальный смысл bento grid, конверсия карточки
🔧 2–3 часа

Skeleton Loading при загрузке

Сейчас при смене категории — мгновенный пустой экран, потом карточки. Нужен shimmer-placeholder в форме карточек: background: gradient + animation: shimmer 1.8s infinite. Staggered delay у разных карточек.

→ Воспринимаемая скорость +40%, профессиональное ощущение
🔧 3–4 часа

Hover-эффекты на карточках

Карточки статичны — нет отклика на hover. Добавить: transform: translateY(-6px) + усиление box-shadow + zoom фото scale(1.05) внутри overflow: hidden. Только на @media (hover: hover) — не ломает мобилку.

→ Воспринимаемое качество продукта +45%
🔧 4–6 часов

Empty State при нулевых результатах

Когда фильтр «Онлайн-запись» даёт 0 результатов — сейчас пустой экран. Нужен: иконка + заголовок + описание («Салонов с онлайн-записью рядом пока мало») + 3 кнопки действий. Особенно важно для фильтров класса B.

→ Доверие пользователя, помощь в навигации
🔧 1 день

Hero — компактный (≤30% viewport)

Hero сейчас занимает ~50% экрана. Категории и карточки не видны без скролла. Уменьшить padding hero, убрать поиск из hero (перенести в sticky bar) или интегрировать поиск прямо в hero-блок как у Fresha.

→ Контент выше fold, CTR по категориям
🏗 2–3 дня

Смена палитры на Warm Mocha

Заменить CSS-переменные в проекте: --cream: #2B241F, --card: #3A3028, --text: #F0EAE3, --accent: #D8956B. Проверить все компоненты. Тёмные карточки не требуют реальных фото — градиент сам по себе красив.

→ Трансформация восприятия бренда
🏗 1–2 дня

Поп-ап при клике на маркер карты

Сейчас клик по маркеру ничего не делает. Нужен mini-card popup над маркером: название + рейтинг + кнопка «Подробнее». При ховере карточки в list-режиме — подсвечивать маркер на карте.

→ Связка карта↔список, engagement на карте
🏗 1 день

Анимации фильтров и категорий

При активации chip — ripple-эффект + scale transition. При смене категории — карточки fade out → fade in (staggered). При первой загрузке — карточки появляются последовательно снизу вверх.

→ Живой, современный feel приложения
