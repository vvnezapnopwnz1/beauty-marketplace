# Задача: Страница мастера в дашборде — drawer-паттерн и полноценный UI

## Контекст

Монорепо `beauty-marketplace`. Дашборд: React + MUI + `useDashboardPalette()`.
Роут `/dashboard/staff/:staffId` уже существует — найди текущий компонент по этому роуту.
Данные мастера приходят с `GET /api/v1/dashboard/salon-masters/:id` (поля: `displayName`, `color`, `isActive`, `masterProfile.bio`, `masterProfile.specializations`, `services[]` с оверрайдами, `status`).
Расписание мастера: `GET /api/v1/dashboard/schedule/staff/:staffId` — возвращает `salon_master_hours[]` (поля: `dayOfWeek`, `opensAt`, `closesAt`, `isDayOff`, `breakStartsAt`, `breakEndsAt`).

Читай `AGENTS.md`. После изменений: `npm run lint` + `npm run build`.

---

## Что нужно сделать

### 1. Улучшить саму страницу мастера

Текущий компонент, вероятно, базовый. Привести к такой структуре:

**Шапка страницы:**
- Кнопка «← Назад» к списку мастеров
- Аватар (круг с инициалами, фон = `master.color`)
- Имя мастера + chips специализаций из `masterProfile.specializations`
- Bio если заполнен
- Бейдж статуса (`active` / `pending` / `inactive`)
- Кнопки: «Редактировать» (открывает существующую `StaffFormModal`) и «Деактивировать» (confirm-модалка)

**Три секции ниже шапки:**

**Секция «Услуги»** — таблица или список:
- Название услуги, базовая цена (из salon_services), override цена если задана (выделить другим цветом), длительность с учётом override
- Кнопка «Настроить услуги» → открывает существующую логику редактирования услуг мастера

**Секция «Расписание»** — компактная таблица 7 дней:
- Каждая строка: день недели, время работы или «Выходной», время перерыва если есть
- Кнопка «Редактировать расписание» → открывает **ScheduleDrawer** (см. ниже)
- Данные загружаются при монтировании страницы через `GET /dashboard/schedule/staff/:staffId`

**Секция «Ближайшие записи»** — последние 5 записей мастера:
- Загрузить через `GET /dashboard/appointments?salonMasterId=:id&limit=5&sort=upcoming`
- Дата, время, клиент, услуга, статус-бейдж
- Клик на строку → открывает **AppointmentDrawer** (см. ниже)

### 2. ScheduleDrawer — редактирование расписания мастера

MUI `Drawer` с `anchor="right"`, ширина `440px` (на мобиле `100%`).

Открывается кнопкой «Редактировать расписание» на странице мастера.

**Содержимое drawer:**
- Заголовок: «Расписание — [Имя мастера]», кнопка закрытия ✕
- Для каждого дня недели (Пн–Вс) — строка:
  - Чекбокс «Выходной» (`isDayOff`)
  - Если не выходной: TimePicker начала и конца (`opensAt` / `closesAt`)
  - Если не выходной: опциональный перерыв — чекбокс «Перерыв» + два TimePicker (`breakStartsAt` / `breakEndsAt`)
- Кнопки внизу: «Сохранить» (PUT запрос) и «Отмена»

**API:** `PUT /api/v1/dashboard/schedule/staff/:staffId` — уже должен существовать. Если нет — добавить в бэкенд: принимает массив из 7 объектов, upsert по `(salon_master_id, day_of_week)`.

Использовать MUI `TimePicker` из `@mui/x-date-pickers` — он уже должен быть в зависимостях (используется в других формах дашборда). Если нет — использовать простой `<input type="time">` обёрнутый в MUI `TextField`.

### 3. AppointmentDrawer — просмотр и редактирование записи

Тот же паттерн: MUI `Drawer anchor="right"` 440px. Открывается кликом на строку записи в секции «Ближайшие записи» на странице мастера, **а также** кликом на строку в `DashboardAppointments.tsx` (заменить там текущую модалку редактирования).

**Содержимое drawer:**
- Аватар клиента (инициалы), имя, телефон
- Бейдж статуса с цветом
- Дата и время (startsAt – endsAt), длительность в минутах
- Услуга и мастер
- Заметка клиента если есть
- Divider
- Форма редактирования (если статус `pending` или `confirmed`):
  - Select мастера, Select услуги, DateTimePicker начала, текстовое поле заметки
- Кнопки действий в зависимости от статуса:
  - `pending` → «Подтвердить» + «Отменить»
  - `confirmed` → «Завершить» + «Отменить»
  - `completed` / `cancelled_*` → только «Закрыть»
- Кнопка «Сохранить изменения» (PUT) если форма редактирования видна

**Создать переиспользуемый компонент** `frontend/src/pages/dashboard/ui/drawers/AppointmentDrawer.tsx`.
Использовать его в `DashboardAppointments.tsx` вместо текущей модалки (или рядом — сохрани модалку для создания новой записи, drawer для редактирования/просмотра существующей).
