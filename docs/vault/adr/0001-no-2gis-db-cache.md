## Контекст

Лицензия 2GIS Catalog API запрещает долговременное хранение сырых каталожных данных (карточки организаций, фото, описания) в нашей БД как замену каталогу.

## Решение

- Запросы к 2GIS **только в реальном времени** при пользовательских действиях.
- В PostgreSQL храним **наши** сущности (`salons`, связи `salon_external_ids`, записи, отзывы и т.д.), не дублируя витрину 2GIS.

## Последствия

- Поиск и карточка места зависят от доступности API 2GIS.
- Нужны fallback и обработка ошибок (unified search → чистый 2GIS и т.д.).

## Альтернативы

- Яндекс / Google Places — другие лицензии; интерфейс `PlacesProvider` оставляет запасной путь.
