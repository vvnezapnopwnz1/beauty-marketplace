# Beautica — Предустановленные категории услуг

> Версия: 2026-04 | Уровень: средний (как в YCLIENTS — папки для группировки услуг)  
> Владелец выбирает категорию из этого списка, затем добавляет свои услуги внутрь со своими ценами и длительностью.

## Scope-модель салона (2026-04-25)

- Вместо единственного `salon_type` для фильтрации категорий используется набор scope-родителей в таблице `salon_service_category_scopes (salon_id, parent_slug)`.
- API профиля салона (`/api/v1/dashboard/salon/profile`) читает/пишет `salonCategoryScopes: string[]`.
- Для обратной совместимости backend использует fallback из `salon_type`, если scopes у салона ещё не заполнены.
- Форма услуги по умолчанию показывает scoped-категории, а полный список остаётся доступен через `allowAllCategories`.

## Структура данных

```sql
CREATE TABLE service_categories (
    id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    salon_id    uuid REFERENCES salons(id) NULL,  -- NULL = системная (общая)
    slug        text NOT NULL,
    name_ru     text NOT NULL,
    parent_slug text NOT NULL,   -- группировка по типу бизнеса
    sort_order  int  NOT NULL DEFAULT 0,
    is_system   bool NOT NULL DEFAULT true
);
```

`salon_id = NULL` → системная, видна всем  
`salon_id = <id>` → кастомная, видна только этому салону (реализуется позже)

## Маппинг: тип салона → доступные категории

При создании/редактировании услуги dropdown показывает только категории, релевантные типу салона. Тип салона задаётся при онбординге в разделе «Профиль».

| Тип салона                              | salon_type slug     | Доступные parent_slug                              |
| --------------------------------------- | ------------------- | -------------------------------------------------- |
| Парикмахерская / Салон красоты (волосы) | `hair_salon`        | `hair`, `packages`                                 |
| Барбершоп                               | `barbershop`        | `barbershop`, `packages`                           |
| Ногтевая студия                         | `nail_studio`       | `nails`, `packages`                                |
| Студия бровей                           | `brow_bar`          | `brows`, `lashes`, `permanent`, `packages`         |
| Студия ресниц                           | `lash_studio`       | `lashes`, `brows`, `permanent`, `packages`         |
| Студия перманентного макияжа            | `permanent_studio`  | `permanent`, `brows`, `makeup`, `packages`         |
| Визажист / Стилист                      | `makeup_artist`     | `makeup`, `hair`, `packages`                       |
| Косметолог                              | `cosmetologist`     | `skin`, `permanent`, `massage`, `packages`         |
| Массажный салон / SPA                   | `spa_massage`       | `spa`, `massage`, `skin`, `depilation`, `packages` |
| Студия депиляции                        | `depilation_studio` | `depilation`, `packages`                           |
| Солярий                                 | `tanning_studio`    | `tanning`, `packages`                              |
| Многопрофильный салон красоты           | `beauty_salon`      | все                                                |
| Частный мастер                          | `individual`        | все                                                |

### Edge cases

- **Многопрофильный салон и частный мастер** — видят все 72 категории. Это осознанный выбор: если мастер делает и волосы, и ногти, ограничивать его нельзя.
- **Тип не выбран** — показывать все категории с подсказкой «Укажите тип салона в профиле, чтобы видеть только нужные категории».
- **Салон хочет добавить категорию не своего типа** — ссылка «Показать все категории» разворачивает полный список. Не блокируем, просто прячем по умолчанию.

## Маппинг на search-категории платформы

**Границы доменов (важно для архитектуры):**

- **Справочник дашборда** (`parent_slug`, slug услуги) — единственный источник истины для того, как владелец группирует услуги в кабинете. Его не нужно подгонять под рубрики внешних API.
- **Маппинг ниже** — отдельная **проекция** «родительская группа дашборда → рубрика поиска на маркетплейсе». Она может меняться (новые фильтры, A/B) без пересборки справочника услуг; в коде это отдельный слой, не смешиваем с таблицей `service_categories`.
- **2ГИС** — отдельный контур: свои рубрики и контракт API. Связь с каталогом салона реализуется **адаптером** (например, `catalog_adapter` в backend), а не через FK или общую таблицу с дашбордом. Категория услуги в дашборде не обязана однозначно соответствовать рубрике 2ГИС.
