# Beautica — Фичи бизнес-дашборда

> Версия: 2026-04-12 | Контекст: Beautica = маркетплейс + SaaS для бизнеса в одном продукте

---

## Философия продукта

Beautica — это не просто CRM для салона. Это **двусторонний маркетплейс**, где клиент ищет салон через поиск, а салон управляет бизнесом через дашборд. Всё, что салон делает в дашборде, напрямую влияет на то, как он выглядит в поиске. Это ключевое отличие от YCLIENTS/DIKIDI (только SaaS) и от Яндекс Карт (только каталог).

**Принцип**: каждое действие в дашборде → улучшение карточки в поиске → больше клиентов → больше записей → больше данных → лучше аналитика.

---

## 1. Публичный профиль салона (карточка в поиске)

Салон управляет тем, как он выглядит для клиентов в результатах поиска и на странице салона.

### 1.1 Основная информация
- Название (NameOverride поверх 2GIS)
- Описание салона (текстовое, до 500 символов)
- Адрес (AddressOverride поверх 2GIS)
- Контакты: телефон, Telegram, WhatsApp, Instagram
- Категория: hair, nails, spa, barber, brows, lashes, massage и т.д.
- Тип бизнеса: venue (заведение) или individual (частный мастер)
- Рабочие часы по дням недели + исключения (праздники, отпуск)
- Геолокация (lat/lng) — подтверждение/корректировка точки на карте

### 1.2 Медиа-контент
- **Главное фото салона** — обложка карточки в поиске (PhotoURL)
- **Галерея фото** — интерьер, рабочие места, оборудование (до 20 фото)
- **Фото работ** — привязанные к конкретной услуге и мастеру (до/после)
- **Видео** — короткие ролики (до 60 сек): процесс работы, результат, обзор салона
- Модерация медиа: автоматическая (NSFW-фильтр) + ручная (жалобы)
- Подсказки при загрузке: «Добавьте фото работ — карточки с фото получают в 3 раза больше кликов»

### 1.3 Влияние на поиск
- Заполненность профиля → коэффициент в ранжировании (чем полнее — тем выше)
- Наличие фото → визуальное превосходство карточки в bento grid (featured card)
- Наличие отзывов и высокий рейтинг → рост в выдаче
- Онлайн-запись включена → бейдж «Онлайн-запись» на карточке, фильтр в поиске

## 2. Услуги

Услуги — это то, что видит клиент на странице салона и по чему может записаться.

### 2.1 CRUD услуг
- Название, описание (опционально)
- Длительность (минуты)
- Цена (рубли; nullable — «по договорённости»)
- Диапазон цен: от–до (например, «Окрашивание: от 3 000 до 8 000 ₽»)
- Категория услуги (группировка: «Волосы», «Ногти», «Брови и ресницы»)
- Сортировка (drag & drop)
- Активность (toggle: показывать/скрыть в публичном профиле)
- Фото работ привязанные к услуге

### 2.2 Привязка к мастерам
- Услуга ↔ мастер (M:N): какие мастера оказывают эту услугу
- У каждого мастера может быть своя цена на услугу (override)
- У каждого мастера может быть своя длительность (например, младший мастер — дольше)

### 2.3 Отображение в поиске
- На карточке в bento grid: до 3 услуг с ценами (для enriched/featured карточек)
- На странице салона: полный каталог услуг с группировкой
- Фильтр в поиске: «Стрижка до 2 000 ₽» → показать только салоны где есть

## 3. Мастера (профили)

Мастера — ключевая сущность. Репутация мастера переносится при смене салона.

### 3.1 Профиль мастера
- Имя, фото (аватар)
- Специализация (теги: колорист, nail-мастер, бровист и т.д.)
- Описание / био (до 300 символов)
- Стаж работы (опционально)
- Сертификаты / обучение (опционально, фото)

### 3.2 Портфолио мастера
- Фото работ (до/после) — привязка к конкретной услуге
- Видео работ — процесс выполнения услуги
- Клиент видит портфолио на странице салона в секции мастера
- При записи клиент может выбрать конкретного мастера по портфолио

### 3.3 Создание профиля
- **Салон создаёт**: владелец добавляет мастера из дашборда (Staff)
- **Мастер создаёт сам**: регистрируется как user с ролью `master`, создаёт профиль, подаёт запрос на привязку к салону
- **Салон подтверждает**: владелец одобряет привязку мастера
- Мастер может быть привязан к нескольким салонам (freelance)
- При уходе из салона — профиль, портфолио и отзывы остаются с мастером

### 3.4 Расписание мастера
- Индивидуальное расписание (может отличаться от салона)
- Выходные, отпуск, больничные
- Синхронизация с общим календарём салона

### 3.5 Отображение в поиске
- На странице салона: список мастеров с фото, специализацией, рейтингом
- Клиент может записаться к конкретному мастеру
- Фильтр: «Мастер с рейтингом 4.5+»
- (Будущее) Отдельный поиск по мастерам, не только по салонам

## 4. Записи и календарь

Ядро операционной работы салона.

### 4.1 Календарь
- Виды: день / неделя / месяц
- Колонки по мастерам (в режиме «день» и «неделя»)
- Цветовая индикация статусов: pending (жёлтый), confirmed (зелёный), completed (серый), cancelled (красный), no_show (тёмно-красный)
- Click на пустой слот → создать запись вручную
- Drag & drop для переноса записи
- Визуальное отображение рабочих часов vs нерабочего времени

### 4.2 Управление записями
- Список всех записей с фильтрами (дата, статус, мастер, услуга)
- Быстрые действия: подтвердить / отменить / завершить / отметить no_show
- Детали записи: клиент (имя, телефон), услуга, мастер, время, заметка
- Ручное создание записи (для записей по телефону / walk-in)
- Создание записи гостем (без регистрации — guest_name + guest_phone)

### 4.3 Статусы записей
```
pending → confirmed → completed
                   → cancelled_by_client
                   → cancelled_by_salon
                   → no_show
```

### 4.4 Расписание и слоты
- Рабочие часы по дням недели (WorkingHour)
- Генерация доступных слотов: рабочие часы − уже занятые записи = свободные слоты
- Учёт длительности услуги при расчёте слотов
- Буферное время между записями (настраиваемое: 0/5/10/15 мин)
- Исключения: праздники, отпуск мастера, технический перерыв

## 5. Отзывы и рейтинг

### 5.1 Отзывы о салоне
- Оставить отзыв может только клиент после завершённой записи (verified reviews)
- Рейтинг: 1–5 звёзд
- Текст отзыва (опционально)
- Привязка к конкретной записи (услуга + мастер)
- Ответ владельца салона на отзыв
- Модерация: спам-фильтр, жалобы, ручная модерация

### 5.2 Отзывы о мастере
- Тот же отзыв привязан и к салону, и к мастеру
- На профиле мастера — отдельный рейтинг и список отзывов
- При уходе из салона отзывы о мастере остаются с ним

### 5.3 Агрегация рейтинга
- `cached_rating` и `cached_review_count` на салоне — обновляются при новом отзыве
- Аналогичные поля на профиле мастера
- Влияние на ранжирование в поиске: высокий рейтинг + много отзывов = выше в выдаче

### 5.4 Управление в дашборде
- Список всех отзывов с фильтрами (рейтинг, дата, мастер)
- Уведомление о новом отзыве
- Быстрый ответ на отзыв прямо из дашборда
- Статистика: средний рейтинг, тренд, распределение по звёздам

## 6. Монетизация: приоритет в поиске и продвижение

### 6.1 Подписка салона
- **Free**: базовый профиль в каталоге, контакты, отображение в поиске
- **Paid**: онлайн-запись, управление расписанием, аналитика, уведомления, расширенный профиль (портфолио, акции), приоритет в выдаче

### 6.2 Буст в поиске
- «Поднять в топ» — платная опция: салон появляется выше в результатах
- Интеграция с bento grid: featured-карточка может быть предоставлена платному салону
- Бейдж «Продвигается» или аккуратная пометка (не агрессивная реклама)
- Оплата: за день / за неделю / за показы (CPI — cost per impression)

### 6.3 Акции и спецпредложения
- Салон создаёт акцию: «Стрижка -20% до конца недели»
- Акция отображается на карточке в поиске (бейдж / текст)
- Акция на странице салона в виде баннера
- Аналитика акции: сколько увидели → сколько записались

### 6.4 Баннерная реклама (будущее)
- Рекламные баннеры внутри результатов поиска
- Таргетинг по категории, району, запросу
- Для косметических брендов и смежных бизнесов

## 7. Аналитика и статистика

### 7.1 Виджеты главного экрана
- Записей сегодня (подтверждённых / ожидающих)
- Новых записей за неделю (% изменения)
- Загрузка мастеров (% занятых слотов)
- Средний рейтинг и количество отзывов
- Выручка за период (когда подключим оплату)

### 7.2 Маркетплейс-аналитика (уникально для маркетплейса)
- **Просмотры карточки в поиске** — сколько раз салон появлялся в результатах
- **Клики на профиль** — сколько раз открывали страницу салона
- **Клики на «Записаться»** — сколько начали процесс записи
- **Завершённые записи** — сколько дошли до конца
- **Воронка конверсии**: показы → клики → записи → визиты
- Эти данные может дать ТОЛЬКО маркетплейс — YCLIENTS/DIKIDI этого не имеют

### 7.3 Аналитика записей
- Загрузка по часам (heatmap: когда пиковые часы)
- Тренд записей за 7/30/90 дней
- Топ-5 услуг по количеству записей и выручке
- Топ мастеров по загрузке и рейтингу
- Процент возврата клиентов (retention rate)
- No-show rate и тренд

### 7.4 Аналитика клиентов
- Всего уникальных клиентов
- Новые vs возвращающиеся (за период)
- Средний чек
- LTV (lifetime value) клиента
- «Спящие» клиенты (не были N дней)

## 8. Уведомления (Telegram-first)

Telegram — основной канал уведомлений (бесплатный API, 98% открываемость в РФ).

### 8.1 Для владельца салона
- Новая запись → мгновенное уведомление
- Отмена записи → уведомление + предложение заполнить слот
- Новый отзыв → уведомление
- Ежедневная сводка: записи на завтра
- Еженедельный отчёт: выручка, загрузка, рейтинг

### 8.2 Для клиента
- Подтверждение записи
- Напоминание за 24 часа
- Напоминание за 2 часа
- Запись завершена → «Как вам? Оставьте отзыв» (ссылка)
- Акция от салона → персональное предложение

### 8.3 Для мастера
- Новая запись к нему
- Изменение/отмена записи
- Расписание на завтра (вечернее напоминание)

## 9. CRM — клиентская база

### 9.1 Профиль клиента
- Имя, телефон (из appointments)
- Количество визитов
- Дата первого и последнего визита
- Любимые услуги (по частоте записей)
- Любимый мастер
- LTV (сумма всех записей)
- Статус: новый / постоянный / VIP / «спящий» (давно не был)

### 9.2 Заметки мастера
- Персональные заметки: формулы красок, аллергии, предпочтения
- Привязаны к клиенту, видны всем мастерам салона
- История заметок по визитам

### 9.3 Автоматическая сегментация
- Новые (1 визит)
- Регулярные (3+ визита)
- VIP (10+ визитов или LTV > порога)
- «Спящие» (не были 30/60/90 дней)
- «Уходящие» (снижение частоты визитов)

## 10. Command Center (проактивные подсказки)

Дашборд не просто показывает данные — он говорит владельцу что делать.

### 10.1 Типы карточек
- 🟡 «3 записи ждут подтверждения» → кнопка «Подтвердить все»
- 🔴 «Отмена: слот 11:00 свободен, 2 клиента в waitlist» → «Предложить слот»
- 💤 «5 клиентов давно не были (60+ дней)» → «Отправить напоминание»
- ⚡ «У Ольги 4 свободных слота в среду» → «Создать акцию»
- ⭐ «2 новых отзыва без ответа» → «Ответить»
- 📸 «Профиль заполнен на 60% — добавьте фото для +40% кликов» → «Загрузить фото»
- 📊 «Загрузка на этой неделе ↓15% vs прошлой» → «Посмотреть аналитику»

### 10.2 Приоритизация
- Карточки сортируются по срочности: pending записи > отмены > спящие клиенты > подсказки
- Максимум 5 карточек на главном экране (не перегружать)

## 11. Уникальные фичи (чего нет у конкурентов в РФ)

### 11.1 Smart Waitlist (авто-замена отмен)
- Клиент хочет записаться, но нет свободных слотов → попадает в waitlist
- Другой клиент отменяет → система автоматически уведомляет первого из waitlist
- «Слот на 11:00 освободился! Записаться?» → Telegram кнопка «Да» / «Нет»
- Первый кто нажал «Да» → получает слот, остальные остаются в очереди
- Timeout 30 мин → предлагается следующему
- Модель WaitlistEntry уже есть в БД

### 11.2 Unified Inbox (Telegram + WhatsApp)
- Все сообщения от клиентов в одном окне дашборда
- Telegram Bot API: клиент пишет боту → сообщение появляется в inbox
- WhatsApp Business API (через провайдера Wazzup/Chat-API)
- Каждое сообщение привязано к профилю клиента в CRM
- Быстрые ответы, шаблоны, отправка ссылки на запись
- Аналог: Mangomint Connect ($75/мес, только США) — мы делаем для РФ бесплатно/дешевле

### 11.3 AI-ассистент «Beautica AI»
- Чат в дашборде: NL-запросы к данным салона
- «Какой мастер загружен меньше всего?» → ответ с рекомендацией
- «Покажи выручку за март» → таблица/график
- «Напиши текст акции на стрижку» → сгенерированный текст
- «Ответь на отзыв от Ирины» → предложенный ответ
- MVP: 5–10 фиксированных типов запросов, обёртка над LLM API + SQL
- Платная фича (paid plan)

### 11.4 No-Show предикция
- Предсказание вероятности неявки по историческим данным
- Сигналы: история no_show клиента, день недели, время, интервал до записи, способ записи
- Действия: низкий риск → обычное напоминание, средний → двойное подтверждение, высокий → предоплата
- MVP: простые правила (клиент с 2+ no_show в истории = высокий риск)
- Позже: ML-модель

### 11.5 Динамическое ценообразование (off-peak скидки)
- Владелец задаёт скидку для непиковых часов
- Система подсказывает оптимальные часы по данным загрузки
- На клиентской стороне: бейдж «🔥 -20% до 12:00» на карточке
- Позиционирование: «умные акции», не «surge pricing»
- Результат: +15–25% загрузка в непиковые часы

### 11.6 Маркетинг-автоматизация (триггерные кампании)
- Автоматические Telegram-сообщения по триггерам:
  - Клиент не был 30 дней → «Скучаем! Скидка 15%»
  - День рождения → поздравление + предложение
  - Первый визит завершён → «Как вам? Оставьте отзыв»
  - Регулярный клиент пора записаться (по среднему интервалу)
- Владелец настраивает триггеры и шаблоны в дашборде
- Аналитика кампаний: отправлено → открыто → записались

### 11.7 Программа лояльности
- Бонусные баллы за визиты (1 визит = N баллов)
- Баллы можно потратить на скидку
- Реферальная программа: «Приведи друга — оба получат скидку»
- Уровни: Bronze → Silver → Gold (по количеству визитов)
- Владелец настраивает в дашборде, клиент видит на сайте

### 11.8 Онлайн-оплата и предоплата
- ЮКасса / СБП — оплата записи онлайн
- Предоплата при записи (снижает no-show на 40–60%)
- Частичная предоплата (депозит)
- Возвраты при отмене (автоматические правила)
- В дашборде: платежи, баланс, история транзакций

## 12. Дополнительные фичи

### 12.1 QR-код салона
- Уникальный QR-код со ссылкой на профиль салона на Beautica
- Салон печатает и размещает в витрине / на стойке / на визитке
- Клиент сканирует → попадает на страницу салона → записывается
- Аналитика: сколько записей пришло через QR

### 12.2 Виджет для сайта и соцсетей
- Embed-виджет «Записаться через Beautica» для сайта салона
- Кнопка записи для Instagram / Telegram-канала / VK
- Мини-форма записи которая открывается по клику

### 12.3 Подарочные сертификаты
- Клиент покупает сертификат на сумму или конкретную услугу
- Отправляет другому человеку (Telegram / ссылка)
- Получатель активирует при записи
- В дашборде: проданные сертификаты, активированные, баланс

### 12.4 Абонементы и пакеты
- Пакет: «5 маникюров по цене 4»
- Абонемент: «Безлимитные укладки за 5 000 ₽/мес»
- Клиент покупает в профиле салона
- Автоматическое списание при записи

### 12.5 Мульти-локации
- Один владелец — несколько салонов
- Единый дашборд с переключением между салонами
- Сводная аналитика по всем точкам
- Общая клиентская база
- SalonMember уже поддерживает это в схеме

### 12.6 Импорт клиентов
- Импорт из Excel/CSV при переходе с другой системы
- Импорт из YCLIENTS / DIKIDI (если есть API/экспорт)
- Маппинг полей при импорте

### 12.7 Экспорт данных
- Экспорт записей в CSV/Excel
- Экспорт клиентской базы
- Экспорт отчётов (PDF)

### 12.8 Логирование действий (audit log)
- Кто и когда создал/изменил/отменил запись
- Кто изменил расписание
- Кто ответил на отзыв
- Полезно для салонов с несколькими администраторами

### 12.9 Роли и права доступа
- Owner: полный доступ
- Admin: всё кроме настроек подписки и удаления салона
- Master: видит только свои записи, своё расписание, может загружать портфолио
- Рецепшн (будущее): видит все записи, может создавать/менять, без доступа к финансам

### 12.10 Быстрая запись через Telegram-бота
- Клиент пишет боту: «Хочу записаться на маникюр в субботу»
- Бот показывает доступные слоты → клиент выбирает → запись создана
- Для салона это дополнительный канал привлечения

### 12.11 Карта загрузки (heatmap)
- Визуальная карта: какие дни и часы загружены, какие свободны
- Помогает владельцу оптимизировать расписание и создавать акции на пустые слоты

### 12.12 Сравнение с конкурентами (benchmarking)
- «Ваш средний чек 2 500 ₽ — это на 15% ниже среднего по району»
- «Ваша загрузка 68% — среднее по платформе 74%»
- Анонимизированные данные по категории и району
- Мотивирует улучшать показатели

### 12.13 Интеграция с социальными сетями
- Автоматический импорт фото из Instagram в портфолио
- Кнопка «Записаться» в профиле Instagram → Beautica
- Шеринг акций салона в Telegram-каналы

### 12.14 Онлайн-консультация
- Клиент может задать вопрос салону перед записью через чат
- «Какой оттенок подойдёт для моего цветотипа?» → мастер отвечает → клиент записывается
- Повышает конверсию из просмотра в запись

### 12.15 Повторная запись (re-booking)
- После завершения визита — предложить записаться на следующий раз
- «Следующий маникюр через 3 недели? Записать на 3 мая?»
- Автоматическое напоминание когда подходит срок
