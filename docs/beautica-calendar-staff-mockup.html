📝 Спецификация UX-паттернов

Ключевые решения, основанные на анализе Fresha, Booksy, YCLIENTS.

Календарь

Режимы отображения

День (основной): колонки = мастера. Высота блока пропорциональна длительности (60px = 1 час). Горизонтальный скролл при >6 мастерах. Как у Fresha и YCLIENTS.

Неделя: колонки = дни. Агрегированные ячейки показывают количество записей по типу. Клик → переход в режим «День». Как Booksy/Fresha.

Месяц: мини-индикаторы загруженности по дням (кругу/полоска). Клик → день.

Взаимодействия

Клик на пустой слот → модалка «Новая запись» с предзаполненным мастером и временем.

Клик на событие → модалка деталей записи (клиент, услуга, статус, действия).

Drag & drop событие → перенос в другой слот или к другому мастеру + подтверждение + уведомление клиенту. Как Fresha.

Drag за нижний край → изменение длительности. Как Fresha.

Zoom (Fresha-стиль): слайдер масштаба — больше записей на экране (Small) vs больше деталей (Large).

Визуальные элементы

Статусы цветом: зелёный=подтверждена, жёлтый=ожидает, красный=отмена, тёмно-красный=no-show, серый=завершена.

Перерывы: штриховка диагональная + текст «Обед» / «Перерыв».

Нерабочее время: диагональная штриховка (как в мокапе). Мастер с выходным — вся колонка затенена.

Красная линия: текущее время (обновляется каждую минуту).

Цвет мастера: каждый мастер имеет свой цвет (7 вариантов), используется в аватарках и опционально в теле события.

Записи

Статусы и переходы

pending → confirmed → completed

pending/confirmed → cancelled_by_client / cancelled_by_salon

confirmed → no_show

Быстрые действия в строке: ✓ подтвердить, ✗ отменить, ✓✓ завершить.

Walk-in: гостевая запись (имя + телефон, без аккаунта). Помечается оранжевым бейджом.

Создание записи

1. Выбор клиента (поиск по имени/телефону, создание нового, или «Гость»)

2. Выбор услуги (из списка с ценами и длительностью)

3. Выбор мастера (только те, кто оказывает эту услугу; неподходящие — серые)

4. Дата + время (показ доступных слотов: рабочие часы − занятые = свободные)

5. Заметка (опционально)

Важно: выбор мастера фильтруется по услуге, а слоты — по мастеру + дате. Как у Fresha.

Услуги

Структура

Группировка по категориям (accordion). Категории из справочника service_categories (72 системных slug + будущий кастом на салон).

Каждая услуга: название, описание, цена (точная или «от»), длительность, привязанные мастера, буфер после, видимость, онлайн-запись.

Override на мастера: мастер может иметь свою цену и длительность для услуги (таблица staff_services). Как Booksy «service variants».

Drag & drop: сортировка услуг внутри категории. sort_order в БД.

Мастера

Карточка мастера

Список: карточки в сетке. Аватар (инициалы + цвет), имя, роль, специализация (теги), загрузка (прогресс-бар), ключевые метрики (рейтинг, визиты, выручка).

Детальная страница: профиль + расписание + мини-календарь на сегодня + список услуг с overrides.

Расписание: по дням недели (opens_at, closes_at). Обед — отдельное поле. Исключения (отпуск, болезнь) — список дат с типом.

Цвет: 7 вариантов для календаря. Выбирается при создании.

Онлайн-запись: toggle «Виден клиентам при онлайн-записи».
