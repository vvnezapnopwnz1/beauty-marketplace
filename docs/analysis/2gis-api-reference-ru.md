# 2GIS Catalog API 3.0: Полный технический справочник

Catalog API 3.0 (`catalog.api.2gis.com`) предоставляет **42+ параметра запроса** на основном эндпоинте `/3.0/items`, поддерживает фильтрацию по нескольким рубрикам через запятую и возвращает данные о рейтинге и отзывах только при явном запросе через параметр `fields`. В этом справочнике описаны все задокументированные параметры, система рубрик, механика полей ответа, пагинация и лимиты — с акцентом на use-case бьюти-индустрии в России.

---

## 1. Параметры поиска: 42 способа фильтрации до получения результатов

Основной эндпоинт `GET https://catalog.api.2gis.com/3.0/items` принимает один обязательный параметр (`key`) и десятки опциональных фильтров.

### Текстовая и типовая фильтрация

Параметр `q` принимает 1–500 символов свободного текста — название компании, категория, номер телефона, даже URL сайта. Он необязателен, если передан `rubric_id`, `building_id` или `org_id`. Параметр `type` ограничивает результаты конкретными типами объектов (через запятую): `branch` — филиалы компаний, `building`, `street`, `station`, `station.metro`, `attraction`, `parking`, `adm_div` (с подтипами: `adm_div.city`, `adm_div.district`, `adm_div.region`), `road`, `route`, `crossroad`, `gate`, `coordinates`. Для поиска бизнесов ключевое значение — **`type=branch`**.

### Шесть режимов сортировки

Доступны через параметр `sort`:

| Значение | Описание |
|----------|----------|
| `relevance` | По умолчанию — учитывает совпадение имени, популярность, рейтинг, расположение и рекламу |
| `distance` | По возрастанию расстояния от `location` или `point` |
| `rating` | По убыванию рейтинга |
| `flamp_rating` | По убыванию рейтинга на платформе Flamp |
| `creation_time` | По убыванию даты создания филиала |
| `name` | Алфавитный порядок по возрастанию |

### Три метода географической фильтрации

Методы взаимоисключающие.

**Круговой поиск** — `point=lon,lat` с `radius` (0–40 000 м при наличии `q`; 0–2 000 м без `q`). Альтернативно: `lon` и `lat` отдельно с `radius`.

**Bounding box** — `point1=lon,lat` (верхний левый угол) и `point2=lon,lat` (нижний правый). Ограничение: диагональ до 2 км без текстового запроса.

**Полигон** — WKT-формат через `polygon=POLYGON((…))`, площадь ограничена ~6 км² без текстового запроса.

Параметр `location=lon,lat` задаёт позицию пользователя для влияния на релевантность **без ограничения области поиска**. `viewpoint1`/`viewpoint2` задают мягкое смещение в сторону области просмотра.

### Фильтры по ID сущностей

| Параметр | Описание | Ограничение |
|----------|----------|-------------|
| `region_id` | ID региона | Часто обязателен как географическое ограничение |
| `city_id` | ID города | До 50 через запятую |
| `district_id` | ID района | До 50, только с `type=branch` |
| `building_id` | ID здания | До 50 |
| `place_id` | ID места | До 50 |
| `subway` | ID станций метро | До 50 |
| `org_id` | ID организации | Одно значение |
| `brand_id` | ID бренда | Одно значение |

### Булевые фильтры качества

`has_photos`, `has_rating`, `has_reviews`, `has_site`, `has_itin`, `has_trade_license` — каждый принимает `true` или `false`. **Отдельного фильтра по цене нет**, хотя ценовые данные могут появиться в полях ответа вроде `booking.offers.price`.

### Временные фильтры

`work_time` принимает значения: `now` (открыто прямо сейчас), `mon,17:00`, `today,09:00`, `fri,alltime` (работает всю пятницу). Параметр `opened_after_date` фильтрует по дате открытия филиала в формате `YYYY-MM-DD`.

### Фильтры по динамическим атрибутам

Шаблон: `attr[<код>]=<значение>`. Доступные коды атрибутов можно узнать, добавив `fields=filters` к любому поисковому запросу и изучив `result.filters.attributes`. Флаговые атрибуты принимают булевые значения (`attr[food_service_business_lunch]=true`); атрибуты-диапазоны — пары min,max (`attr[food_service_capacity]=10,30`). Несколько фильтров по атрибутам комбинируются: результаты должны удовлетворять **всем** условиям.

### Управление поведением поиска

| Параметр | Описание |
|----------|----------|
| `search_type` | `discovery` (по умолчанию), `one_branch` (дедупликация), `indoor`, `ads` |
| `search_nearby` | Булевый, сильно усиливает вес близости |
| `search_is_query_text_complete` | Отключает prefix-matching |
| `search_input_method` | Тип ввода: `keyboard`, `voice` и др. |
| `locale` | Язык ответа: `ru_RU`, `en_AE`, `kk_KZ` и др. |

## 2. Система рубрик: двухуровневая иерархия в отдельном API

**Эндпоинта `/3.0/rubrics` не существует.** Рубрики живут в **Categories API v2.0** по адресу `catalog.api.2gis.com/2.0/catalog/rubric/`, с тремя суб-эндпоинтами:

- `/search` — текстовый поиск по категориям
- `/list` — перечисление категорий по родительскому ID
- `/get` — получение конкретной категории

ID рубрик, найденные здесь, передаются в Places API v3.0 через параметр `rubric_id`.

Иерархия имеет ровно **два уровня**: родительские категории `general_rubric` (широкие области) и дочерние `rubric` (конкретные типы бизнесов). Каждый объект рубрики содержит: `id`, `name`, `alias`, `parent_id`, `type`, `branch_count`, `org_count`.

Чтобы узнать полное дерево: вызвать `/2.0/catalog/rubric/list?region_id=32&parent_id=0` для верхнего уровня, затем углубляться с ID родителя.

### Рубрики бьюти-индустрии для России

Родительская рубрика для всех бьюти-бизнесов — **ID `110543`** («Красота»). Дочерние рубрики:

| ID рубрики | Название | Описание |
|------------|----------|----------|
| **305** | Парикмахерские | Все типы парикмахерских |
| **652** | Салоны красоты / Косметолог | Широкая категория: салоны + косметология |
| **5603** | Ногтевые студии | Маникюр, педикюр |
| **110998** | Барбершопы | Мужские стрижки и уход |
| **56759** | СПА-салоны | Спа и велнес |
| **110816** | Эпиляция | Удаление волос |
| **651** | Солярии / Студии загара | Загар |
| **110355** | Оформление бровей и ресниц | Брови, ресницы |
| **58858** | Визажисты | Макияж |

> Рубрика 652 намеренно широкая — покрывает как «Салоны красоты», так и «Косметолог». Дополнительные категории (массаж, перманентный макияж, обучение) можно найти через Categories API с `parent_id=110543`.

### Использование `rubric_id` в запросах

Параметр принимает **несколько ID через запятую**: `rubric_id=305,652,5603,110998`. Все рубрики в одном запросе должны относиться к одному региону. Географическое ограничение обязательно (`region_id`, `point`+`radius`, `city_id` и т.д.).

**Пример: все бьюти-бизнесы в радиусе 1 км от центра Москвы:**

```
GET https://catalog.api.2gis.com/3.0/items
  ?rubric_id=305,652,5603,110998,56759
  &point=37.624186,55.754285
  &radius=1000
  &type=branch
  &fields=items.reviews,items.schedule,items.point,items.rubrics
  &key=YOUR_KEY
```

## 3. Поля ответа: минимум по умолчанию, богатство по запросу

По умолчанию API возвращает только **`id`, `name`, `type`, `address_name`, `address_comment`**. Всё остальное запрашивается явно через параметр `fields`. Этот механизм одинаков для поиска (`/3.0/items`) и детального запроса (`/3.0/items/byid`).

### Рейтинг и отзывы

Требуют `fields=items.reviews`. Возвращают: `rating` (общий балл, например «4.73»), `review_count`, `general_rating`, `org_rating`, `recommendation_count`, `is_reviewable`, разбивку по тегам. **Тексты отзывов через Places API недоступны** — только агрегированная статистика.

Параметры запроса `has_rating=true` и `has_reviews=true` фильтруют результаты; `sort=rating` сортирует по убыванию рейтинга.

### Фотографии

**URL фотографий не возвращаются.** Поля `items.photos` не существует. Флаг `flags.photos` (через `fields=items.flags`) указывает, есть ли у места фотографии; `has_photos=true` фильтрует по их наличию — но сами ссылки на фото в API отсутствуют. Только рекламные изображения доступны через `external_content[].main_photo_url`.

### Контактная информация

`items.contact_groups` — телефоны, email, сайты — **требует платного разрешения** на ключе сверх стандартной подписки. Возвращает объекты с полями `type` (phone/email/website), `value`, `comment` и расписанием по группам контактов.

### Остальные запрашиваемые поля

| Поле | Описание |
|------|----------|
| `items.point` | Координаты |
| `items.schedule` | Часы работы по дням, флаг `is_24x7` |
| `items.rubrics` | Категории: `id`, `name`, `parent_id` |
| `items.org` | Родительская организация |
| `items.brand` | Бренд |
| `items.full_address_name` | Адрес с городом |
| `items.adm_div` | Административная иерархия |
| `items.description` | Описание |
| `items.flags` | Флаги наличия фото, рейтинга и т.д. |
| `items.attribute_groups` | Динамические атрибуты |
| `items.booking` | Данные для бронирования |
| `items.dates` | Даты создания и обновления |

**Поля с платным доступом:** `items.contact_groups`, `items.floors`, `items.floor_plans`, `items.employees_org_count`, `items.itin`, `items.trade_license`, а также коды ФИАС, ФНС, ОКТМО и ОКАТО.

## 4. Пагинация: максимум 50 на страницу, лимиты зависят от тарифа

Пагинация **постраничная**, не курсорная. Два параметра: `page` (1–1 000 000, по умолчанию 1) и `page_size` (1–50, по умолчанию 20). Каждый ответ содержит `result.total` — общее количество совпадений.

### Ограничения демо-ключа

Демо-ключ (1 месяц, 1 000 запросов суммарно) ограничивает `page_size` до **10** и `page` до **5** — максимум **50 результатов**. Платные ключи снимают эти ограничения (`page_size=50`, `page` до 1 000 000).

### Лимиты запросов

Конкретные ограничения по RPS для облачного API не опубликованы. В документации on-premise упоминается пример **100 запросов в минуту** на ключ. Облачные подписки работают на **месячных квотах** запросов — при превышении ключ блокируется до следующего расчётного периода. Подписки оформляются на 1, 6 или 12 месяцев через Platform Manager на `platform.2gis.ru`.

## Ключевые выводы

1. **Рейтинг и количество отзывов доступны бесплатно** — через `fields=items.reviews`. Это главный сигнал качества, который можно использовать в скоринге на MVP.

2. **URL фотографий недоступны** — используй флаг `flags.photos` только как булевый сигнал для скоринга `profile_quality`.

3. **Телефоны и контакты — платная опция**. На старте без них, достаточно данных 2GIS-профиля.

4. **Демо-ключ не годится для реальных данных** — лимит 50 результатов. Нужен платный ключ ещё до запуска сбора данных о салонах в Москве.

5. **Рубрики — в отдельном API** — перед построением запросов вызвать Categories API для подтверждения актуальных ID.

6. **`work_time=now`** — это тот самый фильтр для фичи «Свободно прямо сейчас». Работает на уровне часов работы, не расписания записей — но как первичный фильтр на стороне 2GIS отлично подходит.
