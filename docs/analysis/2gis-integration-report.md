# Отчёт: проверка интеграции с 2GIS Catalog API 3.0

## Дата и время проверки

**2026-04-04, ~01:08 MSK** (локальное время macOS)

## Окружение

| Параметр | Значение |
|---|---|
| Go | go1.24.0 darwin/arm64 |
| Команда запуска | `go run ./cmd/api` из `backend/` |
| PostgreSQL | контейнер `beauty_db`, порт 5433 (docker compose) |
| HTTP-сервер | `:8080` (по умолчанию из конфига) |

## Запросы

### 1. `q=салон красоты` (Москва, радиус 15 км)

```
GET http://localhost:8080/api/v1/places/search?q=салон+красоты&lat=55.7558&lon=37.6176&radius=15000&page_size=5
```

| Метрика | Значение |
|---|---|
| HTTP-код | **200** |
| `total` | **12 018** |
| Элементов в `items` | 5 (согласно `page_size`) |
| Время ответа | ~2.2 с |

Пример первого элемента:

| Поле | Значение |
|---|---|
| `externalId` | `4504127919796064` |
| `name` | Карибия, развлекательный центр |

### 2. `q=маникюр` (Москва, радиус 15 км)

```
GET http://localhost:8080/api/v1/places/search?q=маникюр&lat=55.7558&lon=37.6176&radius=15000&page_size=3
```

| Метрика | Значение |
|---|---|
| HTTP-код | **200** |
| `total` | **8 176** |
| Элементов в `items` | 3 (согласно `page_size`) |
| Время ответа | ~0.6 с |

## Замечания

- Координаты (`lat`, `lon`) у элементов возвращаются как `0` — 2GIS Catalog API может не включать поле `point` для некоторых типов результатов, либо требуется дополнительный параметр `fields` в запросе. Не является блокером для базовой интеграции.
- Первый запрос выполняется дольше (~2 с) из-за установки TCP-соединения; последующие — быстрее (~0.6 с) благодаря keep-alive.

## Вывод

**Интеграция работает.** API-ключ 2GIS корректно загружается из `.env`, Catalog API 3.0 отвечает HTTP 200, данные парсятся и отдаются клиенту в формате JSON. Основной контракт (поиск, пагинация, `total`, `externalId`, `name`) функционирует штатно.

> **Внимание:** значение `2GIS_API_KEY` в этот файл **не включено** и не должно коммититься в репозиторий.
