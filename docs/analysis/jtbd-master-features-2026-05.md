# JTBD-анализ: фичи для привлечения Мастеров

> Дата: 2026-05-04 | Методология: Jobs to Be Done (Christensen)
> Исключено: всё из `future_plans_postponed.md`

## 1. Job Statement

### Соло-мастер (частная практика / самозанятый)

> «Когда я **строю частную практику в бьюти**, я хочу **управлять записями, клиентами и финансами в одном месте и выглядеть профессионально**, чтобы **привлекать новых клиентов и не тратить вечера на рутину».**

### Мастер в салоне

> «Когда я **работаю в салоне**, я хочу **видеть своё расписание, вести историю клиентов и показывать свои работы**, чтобы **развивать репутацию и зарабатывать больше, а при смене салона не начинать с нуля».**

---

## 2. Три измерения Job

| Измерение | Соло-мастер | Мастер в салоне |
|-----------|-------------|-----------------|
| **Функциональное** | Управлять расписанием, клиентами, финансами из телефона | Видеть своё расписание и записи, вести формулы клиентов |
| **Эмоциональное** | Чувствовать себя «настоящим бизнесом», а не фрилансером в WhatsApp | Чувствовать контроль над своей карьерой и репутацией |
| **Социальное** | Выглядеть профессионально для клиентов (персональная витрина, а не «пишите в директ») | Быть заметным мастером с портфолио и отзывами, а не «одним из» |

## 3. Forces of Progress (см. визуализацию)

### Push (что гонит от текущего решения)
- Хаос записей в WhatsApp/Telegram — двойные букинги
- Портфолио разбросано по Instagram/ВК/Telegram
- Финансы в тетради — нет картины заработка
- Клиенты «забываются» без напоминаний о ребукинге
- Нет потока новых клиентов (только сарафан)
- YCLIENTS от 857₽/мес, DIKIDI — бесплатный, но мало фичей для соло

### Pull (что притягивает к Beautica)
- Маркетплейс = новые клиенты бесплатно
- Профиль мастера переносится между салонами
- Всё в одном: записи, клиенты, финансы, витрина
- Персональная ссылка вместо «пишите в директ»
- Заточено под самозанятых

### Habit (привычки, которые держат)
- «WhatsApp + блокнот хватает»
- Instagram уже используется как портфолио
- Привычка к Excel для учёта
- Если уже на DIKIDI — лень переезжать
- «Нет времени разбираться»

### Anxiety (страхи)
- «Клиенты не найдут меня на новой платформе»
- «Стартап — вдруг закроется»
- «Потеряю данные при переносе»
- «Слишком сложно для одного мастера»
- «Не хочу чтобы салон видел моих личных клиентов»

## 4. Конкурентный ландшафт (Non-Obvious Competition)

Конкуренты — это не только CRM-системы. Мастер «нанимает» разные инструменты на одну и ту же работу:

| Job | Что «нанимают» сейчас | Слабость |
|-----|----------------------|----------|
| Управлять записями | WhatsApp, Telegram, звонки | Хаос, двойные букинги, нет истории |
| Вести клиентскую базу | Контакты телефона, Excel, память | Нет формул, нет напоминаний |
| Показать портфолио | Instagram, ВК | Нет записи, перемешано с личным |
| Считать финансы | Тетрадка, Excel, калькулятор | Нет привязки к услугам, ручной труд |
| Привлекать новых клиентов | Сарафан, Instagram реклама | Дорого, нестабильно |
| Выглядеть профессионально | Визитки, Taplink | Статичное, нет записи |
| Отчитываться для НПД | Приложение «Мой налог» вручную | Нет автоматизации |

**Главный конкурент — non-consumption.** Большинство мастеров вообще не используют CRM, а живут в мессенджерах. Значит, надо не «переманивать с YCLIENTS», а быть проще чем WhatsApp+блокнот.

## 5. Big Hire vs Little Hire

| | Big Hire (регистрация) | Little Hire (ежедневное использование) |
|---|---|---|
| **Триггер** | «Увидела у коллеги красивую ссылку для записи» | «Утро, посмотреть записи на сегодня» |
| **Что важно** | Бесплатность, скорость настройки, красивая витрина | Скорость (< 3 сек), мобильный UX, push-уведомления |
| **Что убивает** | Долгий онбординг, необходимость заполнять 20 полей | Тормозит, неудобно на телефоне, лишние шаги |
| **Метрика** | Conversion: registration → first booking link shared | DAU, sessions per week, bookings per week |

## 6. Приоритизированный список фичей

### TIER 1 — Магниты привлечения (~2-3 недели)

Без них мастер не увидит ценности и не зарегистрируется.

#### 1. Персональная ссылка для записи (booking link)
- `beautica.ru/m/olga-nails` — красивый URL
- Открывает: услуги мастера → выбор слота → запись
- Для Bio в Instagram, подписи в Telegram, визиток
- **Job:** «Дать клиентам один URL вместо переписки»
- **Effort:** ~2-3 дня (роутинг + публичная страница мастера уже частично есть)

#### 2. Мини-сайт / публичная витрина мастера
- Расширение текущей `/master/:id` до полноценного лендинга
- Фото мастера, био, специализации, список услуг с ценами
- Кнопка «Записаться» → переход к выбору слота
- Секция сертификатов (будущее расширение)
- **Job:** «Выглядеть профессионально без создания сайта»
- **Effort:** ~3-4 дня (UI + расширение API)

#### 3. Финансовый трекер мастера
- Автоматический подсчёт дохода с каждой завершённой записи
- Ручное добавление расходов (расходники, аренда, транспорт)
- Дашборд: выручка / расходы / чистая прибыль за период
- Графики по неделям и месяцам
- **Job:** «Понимать сколько реально зарабатываю после расходов»
- **Effort:** ~4-5 дней (новые таблицы + UI + графики)
- **Примечание:** отличается от «аналитика: выручка, загрузка» из postponed — это не аналитика для салона, а персональный P&L мастера

#### 4. Формулы и заметки к клиенту
- При каждом визите: текстовое поле для формулы/заметки
- Привязаны к конкретному визиту + агрегированы на профиле клиента
- Видны при следующей записи этого клиента
- Примеры: формулы окрашивания, аллергии, предпочтения
- **Job:** «Помнить детали клиента спустя месяцы»
- **Effort:** ~2 дня (миграция + UI в drawer записи)
- **Killer-feature для колористов** — этого нет у DIKIDI в бесплатном тарифе

### TIER 2 — Удержание и рост (~2-3 недели)

Делают платформу незаменимой после первой недели.

#### 5. Авто-ребукинг (предложение следующей записи)
- После завершения визита: «Маникюр через 3 нед? Записать на 25 мая в 14:00?»
- Уведомление через Telegram / in-app
- Мастер настраивает интервал по услуге (маникюр = 3 нед, окрашивание = 6 нед)
- **Job:** «Не терять клиентов из-за забывчивости»
- **Effort:** ~3-4 дня (логика + настройки + уведомления)

#### 6. Учёт расходников и себестоимость услуги
- Каталог материалов мастера (название, ед. измерения, цена закупки)
- Привязка расходника к услуге (автосписание при завершении визита)
- Отображение: «Выручка 3 500₽ — Расходники 450₽ = Прибыль 3 050₽»
- **Job:** «Знать реальную маржу каждой услуги»
- **Effort:** ~4-5 дней (новые таблицы + UI)

#### 7. Шаблоны быстрых сообщений клиентам
- Пресеты: «Подтверждаете завтра?», «Жду вас!», «К сожалению, занято»
- Мастер может создавать свои шаблоны
- Отправка в Telegram / копирование в буфер за 1 тап
- **Job:** «Не тратить время на однотипные сообщения»
- **Effort:** ~2 дня (UI + интеграция с Telegram outbox)

#### 8. QR-код мастера
- Генерация уникального QR со ссылкой на витрину
- Скачивание PNG для печати (визитка, стойка, зеркало)
- Аналитика: сколько сканирований → записей
- **Job:** «Превратить оффлайн контакт в онлайн запись»
- **Effort:** ~1 день (генерация QR на фронте, трекинг через UTM)

### TIER 3 — Дифференциация (~3-4 недели)

Чего нет у конкурентов на российском рынке.

#### 9. «Режим аренды кресла»
- Настройка модели: фиксированная аренда / % от выручки
- Авто-расчёт: выручка − аренда − расходники = чистый заработок
- Отчёт за период для арендодателя
- **Job:** «Понимать выгодно ли мне арендовать кресло здесь»
- **Effort:** ~3 дня (расширение финансового трекера)
- **Уникально:** ни YCLIENTS, ни DIKIDI не дают этого мастеру

#### 10. Telegram-бот мастера для записи
- Мастер получает ссылку на «своего» бота (или один бот Beautica с роутингом)
- Клиент: `/start` → видит услуги → выбирает слот → записан
- Inline-кнопки, без ввода текста
- **Job:** «Принимать записи там, где живут мои клиенты — в Telegram»
- **Effort:** ~5-7 дней (расширение существующего бота)

#### 11. Виджет записи для соцсетей
- Embed-кнопка «Записаться» для VK, Taplink, Telegram-канала
- По клику — мини-форма (popup/iframe) с услугами и слотами
- Копирование кода виджета из кабинета мастера
- **Job:** «Принимать записи прямо из моих соцсетей без переходов»
- **Effort:** ~3-4 дня

#### 12. Сертификаты и обучение в профиле
- Загрузка фото дипломов / сертификатов
- Отображение на публичной витрине
- Бейдж «Подтверждённая квалификация» (ручная верификация)
- **Job:** «Показать клиенту что я профессионал, а не самоучка»
- **Effort:** ~2-3 дня (загрузка файлов + UI)

#### 13. Экспорт отчёта для самозанятых (НПД)
- Помесячный отчёт: дата, клиент, услуга, сумма
- CSV/PDF для загрузки в «Мой налог»
- **Job:** «Не мучиться с налоговой отчётностью»
- **Effort:** ~1-2 дня (экспорт из финансового трекера)

#### 14. Режим «Не беспокоить» / управление доступностью
- Закрытие слотов на отпуск/болезнь в 1 тап (диапазон дат)
- На витрине: «Мастер вернётся 15 мая»
- Автоматическая блокировка записи на эти даты
- **Job:** «Спокойно уйти в отпуск без потока сообщений»
- **Effort:** ~2 дня (расширение schedule)

## 7. Рекомендация: стратегия снижения Anxiety и Habit

Фичи — только часть уравнения. Чтобы мастер решился перейти:

### Снижение Anxiety
- **«Попробуй за 2 минуты»** — онбординг с минимумом полей (имя, телефон, 1 услуга → готово)
- **Прозрачная бесплатность** — чёткое «бесплатно навсегда для мастеров» на лендинге
- **Приватность личных клиентов** — явный UI: «Ваши личные клиенты не видны салону»
- **Импорт из Instagram** — парсинг постов как портфолио (future)

### Снижение Habit
- **Booking link работает сразу** — главная ценность доступна в первые 2 минуты
- **Push-уведомления = замена WhatsApp** — мастер получает запись как привычное сообщение
- **Telegram-native** — не заставлять переходить в новый канал, работать там где мастер уже есть

## 8. Метрики успеха (по JTBD)

| Метрика | Что измеряет | Целевое значение |
|---------|-------------|------------------|
| Time to first booking link shared | Скорость Big Hire | < 5 минут |
| Booking link → first client booking | Ценность витрины | > 10% conversion |
| WAU (weekly active users) среди мастеров | Little Hire | > 60% через месяц |
| Bookings per master per week | Реальная польза | > 3 (через месяц) |
| % мастеров использующих формулы | Sticky feature | > 30% (через 2 мес) |
| Financial tracker adoption | Ценность трекера | > 40% (через 2 мес) |
