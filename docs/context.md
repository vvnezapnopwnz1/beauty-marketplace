# Проект: Beauty маркетплейс для России / СНГ

## Суть идеи

Двусторонний маркетплейс салонов красоты и бьюти-мастеров для России и СНГ — аналог **Booksy**, который ушёл с российского рынка после санкций 2022 года.

Платформа решает задачу с двух сторон:
- **Клиент** — находит салоны и мастеров рядом, фильтрует по категории, услуге, рейтингу, дальности, записывается онлайн
- **Салон / мастер** — получает профиль на платформе, управляет записями, получает отзывы и новых клиентов

---

## Анализ рынка

### Почему ниша интересна
- Booksy ушёл из России — прямого аналога с клиентским маркетплейсом нет
- Существующие решения (YCLIENTS, DIKIDI, 1С:Салон красоты) — это **инструменты для бизнеса**, не маркетплейсы для клиентов
- YCLIENTS выпустили YPLACES, но оно работает только если салон уже на YCLIENTS и как самостоятельный маркетплейс не раскручено
- Рынок бьюти-услуг в России огромный — тысячи частных салонов и самозанятых мастеров

### Основные конкуренты
| Продукт | Тип | Проблема |
|---|---|---|
| YCLIENTS | B2B инструмент | Нет клиентского маркетплейса |
| DIKIDI | B2B инструмент | 80k салонов, но не маркетплейс |
| 1С:Салон красоты | B2B инструмент | Сложный, для бизнеса |
| Яндекс Карты | Общий каталог | Не заточен под бьюти, нет онлайн-записи |
| Booksy | Прямой аналог | Ушёл с рынка |

## Модель платформы

### Для клиента (бесплатно)
- Список салонов и мастеров рядом (геолокация)
- Фильтры: категория услуг, конкретная услуга, рейтинг, дальность, цена
- Профиль салона: фото работ, список услуг, цены, отзывы, рейтинг
- Онлайн-запись (если салон подключён как платный клиент)
- Если салон ещё не подключён к платформе — карточка места из 2GIS с контактами и CTA «Позвонить»
- Контакты и связь с салоном (для бесплатных профилей)
- История записей, напоминания

### Для салона — бесплатный тариф
- Базовый профиль в каталоге
- Отображается в поиске
- Клиенты могут оставить отзыв и рейтинг
- Контактная информация

### Для салона — платный тариф (подписка)
- Онлайн-запись клиентов
- Управление расписанием и мастерами
- Аналитика (трафик, конверсия, загрузка)
- Приоритет в выдаче
- Уведомления клиентам (напоминания о записи)
- Расширенный профиль (портфолио, акции)

## Стратегия запуска (решение проблемы курицы и яйца)

### Проблема
Двусторонний маркетплейс: клиенты не придут без салонов, салоны не придут без клиентами.

### Решение
1. **2GIS API** — заполнить каталог автоматически ещё до регистрации салонов (лицензия мягче чем у Яндекс)
2. **Один город** — запуск только в Москве, плотное покрытие, потом масштаб
3. **Базовые профили без участия салона** — салоны появляются в каталоге автоматически, могут "заклеймить" свой профиль бесплатно
4. **Ручной онбординг** — параллельно с разработкой искать первые 5–10 салонов через знакомых / Instagram

### Важно про API
- ⚠️ **Нельзя сохранять данные из API** — только запросы в реальном времени
- Правильная архитектура: реалтайм запрос к 2GIS → показ результатов → своя БД хранит только отзывы/рейтинги/записи привязанные к ID организации
- Яндекс API — более жёсткая лицензия, используем как запасной вариант

### Личный кабинет салона (dashboard)

Отдельный раздел приложения по маршруту `/dashboard` (JWT, доступ через членство в салоне и/или флаг `dashboard_access` у сотрудника). В коде уже есть:

- **Услуги** — CRUD, привязка к системным категориям (`service_categories`, поле `category_slug` у услуги), тип салона `salonType` в профиле для фильтрации списка категорий в форме. Спека slug: **`docs/service-categories.md`**.
- **Мастера (salon-masters)** — API `GET/POST/PUT/DELETE /api/v1/dashboard/salon-masters` (алиасы `/staff/*` для совместимости), вложенный `master_profiles` в ответе, `PUT .../salon-masters/:id/services` для цены/длительности оверрайда на услугу, приглашение по телефону (`GET .../masters/lookup`, `POST .../master-invites`). Подробнее: **`docs/master_profiles_salon_masters.md`**, **`docs/agent-prompt-stage2-master-dashboard.md`**.
- **Публичная витрина мастера (клиент)** — `GET /api/v1/salons/:id/masters` и `GET /api/v1/masters/:masterProfileId` без JWT; страница **`/master/:masterProfileId`** и список мастеров на странице салона (только для нашего UUID-салона). Этап 3: **`docs/agent-prompt-stage3-master-public.md`**.
- **Кабинет мастера (JWT)** — маршрут **`/master-dashboard`** (секции через `?section=`): редактирование своего `master_profiles`, входящие приглашения `salon_masters` со статусом `pending` (принять / отклонить), активные салоны, просмотр своих записей по всем салонам. API: **`/api/v1/master-dashboard/...`** (доступ только если у пользователя есть профиль мастера с `user_id`). При входе по OTP теневой профиль с тем же телефоном закрепляется за аккаунтом (claiming). Подробнее: **`docs/agent-prompt-stage4-master-cabinet.md`**, **`docs/master_profiles_salon_masters.md`**.
- **Записи** — список с фильтрами, смена статусов, создание; карточка существующей записи — правый drawer (`AppointmentDrawer`), открывается кликом по строке в списке записей и с карточки мастера. В JSON записи поле мастера — `salonMasterId` (назначение через `salon_masters.id`).
- **Календарь** — режимы день / неделя / месяц; в дне и неделе события рисуются **таймлайном** по `startsAt` и `endsAt` (высота пропорциональна длительности).

Детальные пути к файлам и идеи следующих задач — в **`docs/status.md` §7** и раздел про dashboard в **`docs/architecture.md`**.

## MVP — что входит

### ✅ Делаем в MVP

**Клиентская часть**
- Поиск салонов по геолокации: гибрид **unified search** (бэкенд `GET /api/v1/search` = 2GIS + матчинг с нашими салонами) с fallback на чистый каталог 2GIS при ошибке
- Список (bento-сетка) + карта, фильтры: категория, сортировка, «онлайн-запись», чипы (часть — в разработке)
- Профиль салона: dual-mode страница `salon/place` (платформенный профиль с записью или 2GIS-витрина с звонком)
- Онлайн-запись (только для платных профилей)
- Регистрация клиента через телефон

**Салонная часть**
- Регистрация и базовый профиль
- Управление расписанием (рабочие часы, слоты) — в дашборде: расписание салона и мастеров, оверрайды дат
- Просмотр и правка входящих записей, календарь с длительностью услуги
- Уведомления о новой записи через Telegram (инфраструктура частично; см. `status.md`)

**Инфраструктура**
- Веб-приложение (React) — мобилка потом
- Go бэкенд + PostgreSQL
- Деплой на VPS

### Контекст: будущее улучшение карты (zoom out)
- При отдалении карты сейчас количество салонов ограничено объёмом уже загруженной выдачи.
- Следующий шаг: сделать отдельный map-driven запрос (по zoom/bounds) и увеличивать `radius` + `page_size` на дальних зумах.
- Добавить более устойчивую кластеризацию и режим `Искать в этой области`, чтобы показывать больше салонов без перегруза UI.

### ❌ Не делаем в MVP
- Отзывы и рейтинги (сложно без аудитории)
- Оплата через приложение (эквайринг = месяц интеграции)
- Аналитика для салонов
- Реклама / продвижение
- React Native приложение

## Уникальные фичи (после MVP)

1. **Профиль мастера ≠ профиль салона** — репутация остаётся с мастером при смене работы
2. **«Свободно прямо сейчас»** — фильтр по доступным слотам сегодня/завтра
3. **Верифицированные отзывы** — только после подтверждённой записи
4. **Самозанятые мастера** — отдельный тип профиля без привязки к салону
