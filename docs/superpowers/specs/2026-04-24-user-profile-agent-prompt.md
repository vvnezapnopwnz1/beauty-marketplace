# Промпт для агента — реализация «Профиль пользователя (/me)»

Скопируй текст ниже и отдай свежему агенту.

---

Реализуй фичу **«Профиль пользователя (`/me`)»** в этом репозитории (Go backend + React/TS frontend).

**Перед стартом прочитай:**
1. `AGENTS.md` — правила работы в репо (engineering + safety + verification checklist).
2. `docs/vault/README.md` и `docs/vault/product/status.md` — актуальное состояние проекта.
3. `docs/superpowers/specs/2026-04-24-user-profile-design.md` — **полная спека задачи, источник правды.** Архитектура, модель данных, API-контракты, фронт-структура, тесты и очерёдность работ — всё там.

**Как исполнять:**
- Иди строго по §8 спеки («Очерёдность работ»), шаги 1→10. Каждый шаг — отдельный коммит с рабочим состоянием репо (не ломай master).
- Перед стартом шага — сформулируй себе мини-план (файлы, тесты), после шага — прогоняй верификацию из `AGENTS.md §4`.
- Соблюдай §9 спеки — что **не** делаем (OTP не трогаем, onboarding-модалку не добавляем, загрузку аватара не делаем и т.д.).
- Отложенные пункты бэклога живут в `docs/vault/product/backlog.md` — **не** реализуй их, но если всплывёт что-то новое «на потом» — дополни этот файл.

**Ограничения:**
- Не коммить и не пушь без явного разрешения пользователя.
- Не запускай деструктивные git-команды.
- Для UI-изменений — запусти dev-сервер и руками пройди golden-path перед «готово» (как указано в §7 спеки).
- Если спека в чём-то противоречит коду или неоднозначна — остановись и уточни у пользователя, не додумывай.

**По завершении каждого шага:** кратко отчитайся (что сделано, какие команды верификации прошли, что осталось). Перед переходом к следующему шагу жди подтверждения.

После шага 10 — обнови `docs/vault/product/status.md` (§10 спеки) и собери финальную сводку для PR.
